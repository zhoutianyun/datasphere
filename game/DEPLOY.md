# 联机版部署说明

## 目标

让 `server.py` 在不同网络下也能访问。

## 方案 1：部署到公网平台

适合：Render、Railway、Fly.io、北岸云服务器等。

### 启动命令

```text
python server.py
```

### 端口说明

`server.py` 会优先读取平台提供的环境变量：

- `PORT`
- `HOST`

所以大多数平台不需要再改代码。

### 部署后

平台会给你一个公网网址，例如：

```text
https://your-game.onrender.com
```

把这个地址发给其他玩家，大家就能在不同网络下加入同一房间。

## 方案 2：本机开服 + 公网穿透

适合：不想部署云平台，只想临时联机试玩。

### 本机先启动

```powershell
python ".\server.py"
```

### 然后做公网穿透

任选一个工具：

- ngrok
- cloudflared tunnel
- frp

工具会给出一个公网地址，把这个地址发给其他玩家即可。

## 注意

- 不同网络联机时，最好让所有玩家都使用稳定网络；
- 如果平台会自动休眠，重新唤醒后房间内存状态会丢失；
- 目前这个版本把房间状态保存在服务进程内存里，适合课堂作业和短时试玩，不适合长期持久对局。
