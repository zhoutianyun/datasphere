# 三机对抗 - 联机房间版

这是一个支持 **开房间、加入房间、不同电脑一起玩** 的 3 Agent 对抗卡牌游戏。

## 这版和前面的区别

- 不是单机网页；
- 需要一台电脑运行 `server.py` 作为房主；
- 其他电脑在同一局域网下，用浏览器打开房主地址即可加入；
- 每位 Agent 在自己的电脑上操作，实时共享同一局牌桌。

## 不同网络能不能打开

可以，但要分两种情况：

- **同一 Wi-Fi / 同一局域网**：直接运行 `server.py` 即可；
- **不同网络**：需要把 `server.py` 部署到公网服务器，或者使用公网穿透工具。

这份文件夹已经补成了**可部署版**，支持通过环境变量设置端口，适合部署到 Render、Railway、Fly.io 一类平台。

如果你想走最简单的公网方案，直接看：

`Render部署-最简步骤.md`

## 快速开始

1. 在房主电脑上打开终端；
2. 运行：

```powershell
python ".\游戏文件\server.py"
```

3. 房主浏览器打开：

```text
http://localhost:8000
```

4. 其他电脑打开：

```text
http://房主电脑的局域网IP:8000
```

例如：

```text
http://192.168.1.23:8000
```

5. 房主先输入名字并创建房间；
6. 另外两位玩家输入名字和房间号加入；
7. 满 3 人后，由房主点击开始游戏。

## 跨网络部署方式

最简单的两种办法：

1. 部署到云平台

- 把 `游戏文件/server.py` 上传到支持 Python 的平台；
- 平台会分配一个公网地址；
- 所有人直接访问这个公网地址即可。

2. 使用公网穿透

- 房主本机运行 `server.py`；
- 再用 `ngrok`、`cloudflared tunnel` 等工具映射到公网；
- 把生成的公网地址发给其他玩家。

## 使用条件

- Python 3.8 或更高版本；
- 3 台电脑在同一 Wi-Fi 或同一局域网；
- 如果 Windows 弹出防火墙提示，请允许“专用网络”访问。

如果要跨网络联机，上面这条“同一局域网”就不再需要，但必须有公网地址。

## 游戏目标

3 个 Agent 像扑克牌一样轮流出牌，谁先打开 2 个宝箱谁获胜。
